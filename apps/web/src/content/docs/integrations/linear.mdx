---
title: Linear integration setup
description: Set up the Faster Fixes Linear integration to automatically create Linear issues from feedback and sync status bidirectionally.
---

Faster Fixes integrates with Linear to automatically create issues when new feedback is submitted and sync status changes between the dashboard and Linear. The integration runs alongside GitHub — you can connect either, both, or neither. For a product overview, see the [Linear integration overview](/integrations/linear).

## Setup

1. Go to **Application Settings → Integrations** from the sidebar
2. Click **Connect Linear** and authorize the Faster Fixes OAuth app on your workspace
3. In **Project Settings → Linear**, link your project to a team
4. Pick the default state, labels, and priority for new issues

The OAuth grant is workspace-wide. Each project then links to a single team within that workspace.

## Auto-create issues

When enabled (on by default), new feedback automatically creates a Linear issue in the linked team. The issue includes:

- Feedback comment
- Page URL
- DOM selector
- Browser info
- Link to the screenshot

Issues are attributed to the Faster Fixes app, not to the member who installed the integration, so they survive that member leaving the workspace.

## Defaults per project

Each project link controls how new issues are created:

| Setting | Description |
|---------|-------------|
| **Team** | The Linear team that receives new issues |
| **Default state** | Workflow state used for new feedback (typically Todo) |
| **Default labels** | Labels applied to every created issue |
| **Default priority** | None, Urgent, High, Medium, or Low |
| **Auto-create** | Toggle automatic issue creation off without unlinking |

If a saved state or label is later deleted in Linear, Faster Fixes falls back to a state of the matching type and surfaces a banner in project settings prompting you to re-pick.

## Bidirectional status sync

Status changes propagate in both directions. Mapping is by Linear's workflow state **type**, not name, so renamed or custom states still work.

| Action | Result |
|--------|--------|
| Feedback marked **resolved** in dashboard | Linear issue moves to a `completed`-type state |
| Feedback marked **closed** (Archived) in dashboard | Linear issue moves to a `canceled`-type state |
| Feedback marked **in progress** in dashboard | Linear issue moves to a `started`-type state |
| Feedback marked **new** in dashboard | Linear issue moves to the project's default state |
| Linear issue moved to **Done / completed** | Feedback status updated to resolved |
| Linear issue **canceled** | Feedback status updated to closed |
| Linear issue moved to **In Progress / started** | Feedback status updated to in progress |
| Linear issue moved to **Triage / Backlog / Todo** | Feedback status updated to new |

## Manual issue creation

You can create a Linear issue manually from any feedback item in the Inbox using the **Create Linear issue** action. The issue is created in a state matching the feedback's current status, so manually linking an already-resolved feedback opens the Linear issue directly in a `completed`-type state.

A feedback can be linked to a GitHub issue and a Linear issue at the same time. Status changes propagate to both.

## Changing teams

Switching a project to a different team requires re-picking the default state and labels in the same step, since IDs are not portable across teams. Faster Fixes opens a full form for this case.

## Unlinking

You can unlink a team from **Project Settings → Linear** at any time. Existing issue links are preserved but sync stops.

## Disconnecting

Disconnecting from **Application Settings → Integrations** removes the OAuth grant and cascades all project links. Existing Linear issues are not deleted; they simply stop syncing.
